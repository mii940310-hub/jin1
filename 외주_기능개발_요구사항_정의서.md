# Highland Fresh (강원도 고랭지 농산물 산지직송 플랫폼) 잔여 기능 개발 요구사항 정의서

## 1. 프로젝트 개요
* **프로젝트명**: Highland Fresh (웹 기반 산지직송 직거래 플랫폼)
* **목적**: 불필요한 유통 마진을 없애고 농민과 소비자를 직접 연결하는 투명한 농산물 직거래 플랫폼 개발
* **현재 기술 스택**: 
  * **Frontend**: Next.js (App Router), React 19, Tailwind CSS
  * **Backend**: Next.js API Routes
  * **Database / Auth**: Supabase (PostgreSQL, Supabase Auth)
  * **결제 연동**: PortOne (포트원) SDK
  * **AI 연동**: OpenAI / Google Gemini API 연동 (일부 연동됨)

## 2. 현재 구현 완료 (As-Is) 상태
외주 개발사는 아래 이미 구현된 코드를 바탕으로 잔여 기능을 추가 개발하고 연동해 주시면 됩니다.
1. **인증 시스템**: Supabase를 활용한 회원가입, 로그인, 비밀번호 찾기 및 권한 분리(Consumer / Farmer)
2. **소비자 프론트엔드 UI**: 메인 페이지, 상품 목록, 상품 상세 스켈레톤, 쇼핑 대시보드(네이버/이마트/쿠팡 실시간 가격 비교 API 연동 완료)
3. **농가(Farmer) 파트너 파트**: 농가 파트너 신청 폼, 가락시장 실시간 도매 시세 조회 API 연동 (`/api/market`)
4. **장바구니 및 결제 (프론트 부분)**: 장바구니 UI 구성 및 PortOne 결제창 호출까지의 프론트엔드 연동 완료
5. **데이터베이스 스키마**: 프로필, 농장, 상품, 장바구니, 주문 관련 테이블(Supabase SQL) 기본 구성 완료

---

## 3. 잔여 개발 및 외주 요청 범위 (To-Be)
본 견적 요청의 핵심이 되는 개발 요구사항입니다.

### 3.1. 결제 / 주문 시스템 백엔드 고도화 (Consumer 쪽 완료 작업)
* **결제 검증 API 구현**: PortOne 결제 성공 후, 백엔드에서 결제 금액 위변조 여부 검증 및 승인 로직 구현 (`/api/payment/verify`)
* **주문 데이터 DB 연동**: 결제 검증 성공 시 `orders` 및 `order_items` 테이블에 주문 내역 저장
* **장바구니 초기화 처리**: 주문이 완료된 상품은 소비자의 장바구니(`carts`, `cart_items`)에서 삭제 처리 로직
* **소비자 마이페이지 (주문 내역)**: 본인의 주문 내역, 결제 상태, 배송 상태(상품준비중/배송중/배송완료 등)를 조회할 수 있는 UI 표출

### 3.2. 농가 파트너 관리자 페이지 (Farmer Dashboard)
* **상품 관리 (CRUD)**: 
  * 판매할 상품 등록, 수정, 삭제 기능 개발
  * 썸네일/상세 이미지 업로드 (Supabase Storage 활용) 
  * 단가, 물류비, 플랫폼 수수료, 재고 수량(Stock), 수확일 등 입력 폼 연동
* **주문 및 배송 관리**: 
  * 소비자가 결제한 본인 농가의 주문 내역 리스트 확인
  * 주문 상태 변경 로직 및 운송장 번호(택배사 연동) 입력 기능
* **정산 관리 화면 (추가 검토 사항)**: 농가별 판매 대금 누적 현황, 수수료 차감 후 정산 예상 금액 등 조회 UI

### 3.3. 총괄 관리자 페이지 (Super Admin Dashboard)
* **회원 및 농가 관리**: 가입된 전체 회원 관리 및 **농가 가입 신청 승인/반려** 처리 기능
* **전체 주문 모니터링**: 플랫폼 내 표출되는 모든 주문/배송 현황 전체 조회 (CS용)
* **플랫폼 수수료 관리**: 상품별 혹은 전체 기본 수수료율 일괄 조정 기능

### 3.4. AI 기능 로직 고도화 및 부착
* **AI 적정가 추천 (농가용)**: 농가가 새로운 상품을 등록할 때, Gemini/OpenAI 및 시장 도매가 데이터를 연계하여 최적의 소비자 판매가를 제안(`Price Recommendation`)해 주는 기능 완성
* **AI 고랭지 채소 전문 챗봇 (소비자용)**: 
  * 소비자용 상품 상세 페이지 혹은 메인 페이지 하단에 붙는 특화 챗봇 연동
  * 농산물 보관법, 요리 추천 등을 응대하고 구매를 유도하는 프롬프트 적용 완료된 상태를 UI와 매끄럽게 연결

### 3.5. 상품 검색 및 리뷰 시스템
* **검색 및 필터링**: 카테고리(채소/곡물), 농장명, 상품명, 가격 범위 기반 검색 및 페이지네이션(Pagination) 도입
* **구매자 리뷰/별점 (추가 검토 사항)**: 배송 완료 후 소비자의 리뷰 작성, 사진 첨부 시스템 개발 및 DB 구축

---

## 4. 외주 업체 참고 사항
* 모든 개발은 기존의 **Next.js App Router** 기반 아키텍처 패턴을 따르며, **Supabase DB/Auth** 생태계 내에서 구현하는 것을 원칙으로 합니다.
* PC뿐만 아니라 모바일 환경을 고려한 **반응형 컨테이너(Responsive Web)** 디자인 적용이 필요합니다.
* 개발 후 AWS, Vercel 등을 통한 배포(Deployment) 지원이 포함되는지 견적에 포함해 주시기 바랍니다.
